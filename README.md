# Ultramarine

# Архитектура проекта

Проект разделён на фронтенд и бэкенд.

## Структура проекта

```
/frontend          # фронтенд-приложение (планируется на Vue)
/backend
  /src
    /config        # конфигурации проекта (БД, порты, внешние сервисы)
    /controllers   # обработка HTTP-запросов, вызов сервисов
    /dtos           # Data Transfer Objects (DTO) — структуры данных для передачи между слоями
    /models         # ORM-модели и схемы данных
    /middleware    # middleware (аутентификация, логирование, валидация)
    /routes        # маршруты API, связывают пути с контроллерами
    /service       # бизнес-логика, CRUD-операции
  app.js           # сборка и конфигурация Express-приложения
  server.js        # точка входа, запуск приложения
```

### Пояснения

* **frontend/** — пока пустая папка для будущего Vue-приложения.
* **config/** — все конфигурации (например, база данных, переменные окружения, внешние сервисы).
* **controllers/** — получает HTTP-запросы, вызывает сервисы и возвращает ответ.
* **dtos/** — Data Transfer Objects. Пример DTO для пользователя:
* **models/** — каждая сущность лежит в отдельном файле, связи между моделями определены в `index.js`. Примеры моделей:

```javascript
const User = sequelize.define('user', {...});
```

Связи моделей определены централизованно в `models/index.js`. Примеры:

* User ↔ Student / Employee (One-to-One)

* Employee ↔ Role (Many-to-One)

* Department → Employees (One-to-Many)

* Group ↔ Students (One-to-Many)

* Group ↔ Discipline (Many-to-Many через GroupDiscipline)

* User ↔ Token (One-to-One)

* **middleware/** — промежуточные обработчики запросов, например `authMiddleware`.

* **routes/** — маршруты API для каждой сущности. Все маршруты объединены в общий `routes/index.js` и подключают middleware для защиты (например, `authMiddleware`).

### Список маршрутов и ограничения

**/api/user** (доступ: открыт)

* POST /           — создать пользователя
* GET /            — получить всех пользователей
* GET /:id         — получить пользователя по ID
* PUT /:id         — обновить пользователя
* DELETE /:id      — удалить пользователя
* POST /full       — создать пользователя с дополнительными данными

**/api/student** (доступ: `authMiddleware`)

* POST /           — создать студента
* GET /            — получить всех студентов
* GET /:id         — получить студента по ID
* PUT /:id         — обновить студента
* DELETE /:id      — удалить студента

**/api/employee** (доступ: `authMiddleware`)

* POST /           — создать сотрудника
* GET /            — получить всех сотрудников
* GET /:id         — получить сотрудника по ID
* PUT /:id         — обновить сотрудника
* DELETE /:id      — удалить сотрудника

**/api/auth** (доступ: открыт, кроме logout и refresh)

* POST /login      — вход пользователя

* POST /logout     — выход (требует `authMiddleware`)

* GET /refresh     — обновление токена (требует `authMiddleware`)
